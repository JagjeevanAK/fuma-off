# Open Food Facts API v2

The original Open Food Facts API provides comprehensive access to the world's largest collaborative food database. This mature API offers reliable endpoints for accessing product information, nutrition data, images, and more.

<Callout className="prose" type="info">
API v2 is the stable, production-ready API that powers many applications and services. It provides backward compatibility and extensive functionality.
</Callout>

## Base URL

All API v2 requests should be made to:

```
https://world.openfoodfacts.org
```

## Key Features

- **Comprehensive Data**: Access to nutrition facts, ingredients, images, and metadata
- **Multi-language**: Support for products and data in multiple languages
- **Image Management**: Upload, crop, and manage product images
- **Search Capabilities**: Advanced search with filters and sorting options
- **Batch Operations**: Efficient handling of multiple product operations

<Callout type="success">
Most read operations are public and don't require authentication. Write operations may require user credentials.
</Callout>

## Common Use Cases

- **Product Lookup**: Get detailed information using barcodes or product IDs
- **Nutrition Analysis**: Access comprehensive nutrition data for dietary applications
- **Image Processing**: Upload and manage product photos for better data quality
- **Data Contribution**: Add or update product information to help the community

## Getting Started

The most commonly used endpoint is [Get Product by Barcode](/docs/v2/read-requests/get-product-by-barcode), which provides detailed product information using a standard barcode scan.

<Callout type="warn">
Always include `json=1` parameter to receive responses in JSON format. Without it, responses will be in HTML format.
</Callout>
